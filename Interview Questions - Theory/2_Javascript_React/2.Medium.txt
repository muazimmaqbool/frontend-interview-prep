1. What are closures?
    Answer:
        A closure allows a function to access variables from its outer/parent scope even after the outer function has finished execution.

        function outer(){
        let count=0;

        return function(){
                count++;
                return count;
            };
        }
        let counter=outer();
        counter()
        counter()

        Closures are used for data privacy and maintaining state.

2. What is event delegation?
    Answer:
        Event delegation attaches a single event listener to a parent element instead of multiple child elements.
        It improves performance and works well with dynamically created elements.

4. Difference between synchronous and asynchronous code?
    Answer:
        Synchronous code executes one line after another.

        Asynchronous code allows tasks to run without blocking the main thread.
        Examples include:
            setTimeout
            API calls
            Promises

5. What is the Event Loop?
    Answer:
        The event loop continuously checks the call stack and callback queue. When the stack becomes empty, it moves queued tasks for execution.
        It enables JavaScript to perform asynchronous operations.
                        or
        Event Loop is a core mechanism within the javascript runtime environment that enables asynchronous behaviour, 
        non-blocking programming while running on a single thread. It continuously monitors the execution state and coordinates the processing of code
        between the call stack, background APIs, and task queues.

6. Difference between call(), apply(), and bind()?
    Answer:
        All three control the value of 'this'
            call() accepts arguments individually.
            apply() accepts an array.
            bind() returns a new function.

        The core difference is that call() and apply() invoke a function immediately, whereas bind() returns a brand new function with a pre-set context to be executed later.

8. Difference between slice() and splice()?
    Answer:
        slice() creates a copy without modifying the original array.
        splice() changes the original array.
        
        ->slice() method is a built-in function used to extract a portion of an array or a string and return it as a new entity without modifying 
          the original data. Because it keeps the source data intact, it is considered a non-destructive or immutable operation
        Syntax: sequence.slice(start, end)

        ->splice() method in JavaScript is a built-in function used to modify an array by removing, replacing, or adding elements in place
            Splice syntax:
                const fruits = ['Apple', 'Banana', 'Cherry', 'Date'];
                const removed = fruits.splice(1, 2); 

                console.log(fruits);  // ['Apple', 'Date']
                console.log(removed); // ['Banana', 'Cherry'] (Returns the deleted items)


        ->Checkout code and example of slice and splice in js repo

9. What is debounce?
    Answer:
        Debouncing delays execution until a certain amount of time has passed.

        Common use cases:
            Search bars
            API calls
            Input validation

        It prevents unnecessary function executions.

10. What is throttling?
     Answer:
        Throttling limits how often a function executes.

        Used for:
            Scroll events
            Resize events
            Mouse movement

        ->Throttling is a performance optimization technique that limits how often a function can execute over a specific period. 
         It ensures that a function runs at most once within a designated time window, ignoring any additional triggers until that interval ends

15. Difference between Context API and Redux?
    Answer:
        Context API is suitable for small global states. (Its builtin)
        Redux is preferred for large applications requiring predictable and centralized state management.
