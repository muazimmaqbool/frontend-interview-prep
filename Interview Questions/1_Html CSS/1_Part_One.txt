1: Difference between display: none and visibility: hidden?
   ANS: 
    display: none: This completely removes the element from the document flow. 
                    It is as if the element does not exist on the page; other elements will move to fill the empty space.

    visibility: hidden: This hides the element visually, but it still occupies its original space in the layout. 
                        The element remains in the document flow, resulting in an empty or "blank" gap where the element would normally be

    display: none → removes from layout
    visibility: hidden → keeps space

2: What is Flexbox vs Grid? When would you use each?
   ANS:
    Flexbox → 1D layout
    Grid → 2D layout
    Your Example: “I use Flexbox for components and Grid for full dashboard layouts.”

    ->Flexbox (Flexible Box Layout) is for one-dimensional layouts (a single row or column), 
      while CSS Grid is for two-dimensional layouts (both rows and columns simultaneously).
      Think of Flexbox as content-driven (expanding to fit text/items) and Grid as layout-driven (defining the rigid structure first).

   ANS:
    Responsive design is a web design approach where a website automatically scales, shrinks, or reorganizes its layout to fit perfectly 
    on any device—whether it's a mobile phone, tablet, or desktop
    We can implement it using:
        Media queries

4. Difference Between Relative, Absolute, Fixed, and Sticky Positioning?
   ANS:
      ->Relative: Position relative to itself.
                  Eg: 
                     .box {
                           position: relative;
                           top: 10px;
                        }
                  
      ->Absolute: Position relative to nearest positioned parent.
                  Eg:
                     .box {
                           position: absolute;
                           top: 0;
                        }

      ->Fixed: Position relative to viewport.
               Eg:
                  .navbar {
                           position: fixed;
                           top: 0;
                        }

      ->Sticky: Acts relative until scroll threshold.
                Eg:
                  .header {
                           position: sticky;
                           top: 0;
                        }

5. Explain CSS Specificity?
   ANS:
      CSS Specificity is a set of rules browsers use to determine which style rule "wins" when multiple rules target the same HTML element. 
      It is essentially a ranking system where more specific selectors carry more "weight" than general ones

      ->Priority: Inline Styles, ID, Class / Attribute / Pseudo-class, Element

      ->Eg: #title {
               color: red;
            }
            .title {
               color: blue;
            }
         Result: red (because ID has higher specificity then class)

6. What is Box Model?
   ANS:
      The CSS box model is a fundamental concept where every HTML element is treated as a rectangular box. It determines how an element's total size 
      and its spacing relative to other elements are calculated.
      Every element consists of:
                     Content
                     Padding
                     Border
                     Margin
      Visual:
         Margin
            └ Border
               └ Padding
                  └ Content

7. Difference Between content-box and border-box?
   ANS:
      The main difference between content-box and border-box is how the width and height of an element are calculated in relation to its padding and border.

      -> Default: box-sizing: content-box;
         Width excludes padding and border.
      
      ->Preferred:
         box-sizing: border-box;
          Width includes padding and border.
         Most projects use:
         * {
             box-sizing: border-box;
            }

9. Difference Between em, rem, %, vw, and vh?

ANS:
These are CSS units used for sizing elements, fonts, spacing, etc.

   em → Relative to the parent element's font size.
   rem → Relative to the root (html) font size.
   % → Relative to the parent element.
   vw → Relative to the viewport width (1vw = 1% of viewport width).
   vh → Relative to the viewport height (1vh = 1% of viewport height).

# Example
   html {
   font-size: 16px;
   }

   .parent {
   font-size: 20px;
   }

   .child {
   font-size: 2em;   /* 40px */
   }

   .title {
   font-size: 2rem;  /* 32px */
   }

   .box {
   width: 50%;
   }

   .hero {
   height: 100vh;
   }

Interview Summary:

   em → parent-based
   rem → root-based
   % → parent-based sizing
   vw/vh → viewport-based sizing


10. Difference Between inline, block, and inline-block?

ANS:
   These determine how elements behave in the document layout.

Block:
   Takes full available width.
   Starts on a new line.
   Width and height can be set.

Examples: div, p, section

div {
  display: block;
}

Inline:
   Takes only required width.
   Does not start on a new line.
   Width and height cannot be applied properly.

Examples: span, a, strong

span {
  display: inline;
}

Inline-Block:
   Stays inline like text.
   Width and height can be set.
   button {
   display: inline-block;
   width: 150px;
   }

Interview Summary:

   block → New line + full width
   inline → Same line + content width
   inline-block → Same line + custom width/height
